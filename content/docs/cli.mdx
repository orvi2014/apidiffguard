---
title: CLI
description: Run apidiff check locally or in CI against JSON files or a live URL.
---

Use `@apidiffguard/cli` to fail a pipeline when a response drifts from a known-good baseline.

## Install from this repository

```bash
# Build the diff engine, then install the CLI package
cd packages/diff-engine && npm run build && cd ../..
cd packages/cli && npm install
```

The binary is `apidiff` (also available via `npm run apidiff -- …` inside `packages/cli`).

npm publish of `@apidiffguard/cli` is optional — clone/link from source works today.

## Check two JSON files

```bash
apidiff check --baseline baseline.json --current live.json --fail-on breaking
```

## Check a baseline against a live URL

```bash
apidiff check --baseline baseline.json --url https://api.example.com/v1/me --fail-on breaking
```

## CI tip

Exit code `1` means the threshold was hit. Wire it after deploy smoke or contract fixtures:

```yaml
- run: npx --yes --package=@apidiffguard/cli apidiff check --baseline ./fixtures/users.json --url "$API_URL/users" --fail-on breaking
```

(When the package is published to npm; until then run from `packages/cli`.)

## Options

| Flag | Description |
| --- | --- |
| `--baseline` / `--old` | Path to baseline JSON |
| `--current` / `--new` | Path to current JSON |
| `--url` | Fetch current JSON from an HTTPS URL |
| `--fail-on` | `breaking` (default), `warning`, or `info` |
| `--json` | Print structured JSON instead of text |

Cloud console login (`apidiff login`) and hosted endpoint IDs remain on the roadmap — use the [console](/dashboard) for scheduled monitoring today.
