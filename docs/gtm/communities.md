# Communities — where to help (not spam)

Be useful first. Mention APIDiffGuard only when it directly answers the question.

## Places

| Place | Angle |
|---|---|
| r/webdev, r/programming, r/devops | Answer "API changed and broke us" threads; link free JSON Diff when comparing payloads |
| r/saas, Indie Hackers | Share the free tools + how you use baselines for integrations |
| Dev.to | Publish the blog posts; reply to API/CI questions with concrete steps |
| Hacker News | Show HN once the free tools + OSS package are solid — lead with the free/OSS value |
| API / Stripe / Shopify Discords | Help with webhook/payload debugging; offer baseline monitoring when recurring |

## Reply formula

1. Answer the person's problem in plain language (2–5 sentences).
2. Give a free actionable step (paste into JSON Diff, ignore `request_id`, etc.).
3. Optional soft mention: "We productized this as continuous monitoring at …"

## Don't

- Drop a link with no help
- Cross-post the same launch spam everywhere the same day
- Argue about pricing in the first reply
